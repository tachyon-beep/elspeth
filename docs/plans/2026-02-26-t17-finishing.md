**Step 3: Fix known signature mismatches**

Two of the three `_TestablePluginContext` subclasses have `record_validation_error()` overrides that are missing the `contract_violation` keyword argument:

- `tests/integration/plugins/sources/test_contract.py:38-61` — missing `*, contract_violation: Any | None = None`
- `tests/integration/plugins/transforms/test_contract.py:45-68` — missing `*, contract_violation: Any | None = None`

The third file (`test_trust_boundary.py`) uses `**kwargs` which is already compatible.

Add the missing keyword argument to both overrides to match the `SourceContext` protocol signature exactly:
```python
def record_validation_error(
    self, row, error, schema_mode, destination, *, contract_violation=None
) -> ValidationErrorToken:
```

> **[R5] Review amendment:** Do NOT use `**kwargs: object` as a shortcut here, even though
> `test_trust_boundary.py` uses that pattern. `**kwargs` creates a permanent mypy blind spot:
> if the base `record_validation_error()` signature changes in the future, mypy will NOT flag
> the subclass override as mismatched. Matching the exact signature is the only safe approach.
